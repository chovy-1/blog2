关于博客搭建的心得

Created2024-08-07|Updated2024-08-07
|Post Views:

先说结论：很困难（或许是因为在暑假，想静下心来钻研一个新事物还是不容易的）对于搭建博客 我之前也只是有所耳闻，本来觉得按照教程一步步走就可以了，但是嘛，还是遇到了不少问题的。主要问题以及心得如下： 1.计算机的命令提示符 非管理员权限运行安装命令导致无法删除某些目录 使得安装过程受阻 需要重新以管理员身份操作 我一开始认为问题出在32字节 有些操作似乎也会报错 搞定这些问题消耗我很多时间 我一开始遇到的问题是 创建的快捷方式在属性中设置到我创建的博客目录下 但是以管理员身份运行就会回到默认的c盘目录 所以我去查找了cd切换目录的方法 这也是我遇到的第一个问题 2.最烦的就是报错 拿安装 hexo-deployer-git 这个插件来说 原本以为只是一个简单的命令 npm install hexo-deployer-git –save 就能解决 可实际操作时却遇到了诸多问题 类似这种安装问题太多了 每当报错 就得上网去查找解决方案 其中的信息还得我自己去排除辨别 挺考验心态的 3.我的知识还是太匮乏了 第三方源这个东西 我是不太了解的 比如我在尝试从淘宝源安装 cnpm 时 遇到了证书过期的错误 npm error code CERT_HAS_EXPIRED（应该是这个？） 4.我是在b站上搜的教程 遇到一些细节上的问题 再去询问AI 经过实操 我认为尽管AI存在少数误导情况 但AI问答的效率明显高于学习一般的教程 包括一些概念和术语的理解上 AI基本可以快速准确地回答我的问题 5.我虚拟机也搭完了 给我的感觉就是计算机的实操性很强 仅依靠理论的学习完全是不够的 而且在实操的过程中发现问题并解决 是可以进步很快的 6.多捣鼓捣鼓机子 还是很有必要的（比如说32位和64位的区别、配置环境和选择软件、YAML格式中引号的作用等 如果理解有误会造成许多不必要的麻烦 本人计算机小白 若是有写的不到位的 劳烦大佬们指正（总结在开头
